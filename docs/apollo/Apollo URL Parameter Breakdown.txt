There are some URL parameters in the Apollo filtering that are using some kind of tag IDs or IDs instead of the actual filter value. Here’s the information. We need to build some logic to parse them in our filtering process.




Parameter
	Example Value
	What the ID Represents
	organizationIds[]
	57c4ace7a6da9867ee5599e7
	Organization unique ID (internal Apollo/company ID)
	notOrganizationIds[]
	5f2a39cb77a7440112460cf5
	Organization unique ID to exclude
	organizationIndustryTagIds[]
	5567cd4773696439b10b0000
	Industry tag ID (not a plain text industry name)
	organizationLatestFundingStageCd[]
	0, 1, 2, 10, etc.
	Funding stage codes (IDs for funding stages)
	

Notes
* Arrays in parameters use [] and can have multiple values.
* All fields support URL encoding (e.g., spaces as %20, ampersands as %26).
* IDs are internal Apollo database references for organizations and industries.
* For company keyword matching/exclusion, related fields determine in which metadata (e.g., tags, name, social media description) the operation applies.
* All filters are optional; omit unused ones for broader results.

Subfilter Example
* Umbrella: Job Titles
   * Include: personTitles[]=manager, personTitles[]=consultant
   * Exclude: personNotTitles[]=teacher
* Umbrella: Companies
   * Include: organizationIds[]=...
   * Exclude: notOrganizationIds[]=...
* Umbrella: Company Keywords
   * ANY include: qOrganizationKeywordTags[]=..., field: includedOrganizationKeywordFields[]
   * ALL include: qAndedOrganizationKeywordTags[]=..., field: includedAndedOrganizationKeywordFields[]
   * ANY exclude: qNotOrganizationKeywordTags[]=..., field: excludedOrganizationKeywordFields[]
